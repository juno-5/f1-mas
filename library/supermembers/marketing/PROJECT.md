## 업무 구조: 가설 → 실험 → 검증

### 현재 가설 목록

| # | 가설 | 실험 방법 | 검증 지표 | 상태 |
|---|------|---------|---------|------|
| H1 | "퀄리티 구조 설명"(등급 차등 메커니즘)이 단순 후기보다 CPL 낮다 | 카피A(퀄리티) vs 기존 후기형 | CPL | 🟡 대기 |
| H2 | "배민 비유" 자동화 설명이 이해도 높여 CTR 올린다 | 카피B(자동화) A/B | CTR, 완주율 | 🟡 대기 |
| H3 | 수치 직접 대비(인당 1.5만 vs 5만)가 클릭 유발한다 | 카피C(수치) vs 기존 | CTR | 🟡 대기 |
| H4 | 감정 서사(폐업 위기)가 CPL을 낮춘다 | 카피E(폐업) vs 정보형 | CPL | 🟡 대기 |

### 실험 우선순위
1. H4 — 폐업 서사 (CPL 직접 개선, 제작 쉬움)
2. H1 — 퀄리티 구조 설명 (CTR + CPL 동시 기대)
3. H2 — 배민 비유 자동화 (신규 각도)
4. H3 — 수치 직접 대비 (검증된 수치형 + 신규 각도)

## 소재 제작 파이프라인

```
1. 가설 선택
        ↓
2. 인물 이미지 생성 (Nano Banana Pro / Grok)
   → ai-image-tips.md 참조
        ↓
3. 영상 생성 (Veo 3.1 JSON or Kling AI)
   → production-ready-scripts.md 의 JSON 프롬프트 사용
   → video/ai-video-automation-workflow.md 참조
        ↓
4. 후반 작업 (자막 + 텍스트 오버레이 + BGM + Topaz 4K)
        ↓
5. Meta 업로드 (네이밍: YYMMDD-유형-업종-컨셉-v1-LNK-VID)
        ↓
6. 48시간 후 CTR 확인
   CTR > 5% → 예산 확대
   CTR < 2% → 중단 + 다음 베리에이션
```
