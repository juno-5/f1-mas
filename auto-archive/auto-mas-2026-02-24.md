## Cycle #11 — Sales Domain False Positive: "하이브리드" Regex Leak

### [관찰]
- Request `cdde93f1` ("React 19 서버 컴포넌트...하이브리드 아키텍처...상태관리") selected Signal (SLS-06, Sales) for a pure frontend dev query
- Domain detection: `domains=['developers', 'sales']` — sales incorrectly detected
- Root cause: `리드.*관리` pattern matched "하이브**리드** 아키텍처를 설계해줘. 상태**관리**" — `.*` spans 28+ chars across sentences
- Same issue with "하이브리드 클라우드 관리" → false sales match

### [가설]
- **Claim**: Tightening `리드.*관리` to `리드.{0,4}관리` prevents cross-sentence matches
- **Basis**: Real "리드 관리" has 0-3 chars between words; false matches have 28+ chars
- **Prediction**: React query → developers only; "리드 관리 파이프라인" → still sales
- **Null**: Legitimate queries with >4 chars gap would be missed
- **Risk**: low

### [실험 결과]

|-------|-----|-----|----------|
| React 하이브리드...상태관리 | True ❌ | False ✅ | False |
