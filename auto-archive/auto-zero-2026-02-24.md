4. 레버리지 포인트: Guard in `load()` prevents clearing when entries is empty; guard in `_load_functions_yaml()` prevents caching empty patterns.

**[가설]**
- Claim: "no suitable personas found" caused by hot reload clearing index during SCP deployment
- Basis: (a) `load()` clears dicts even when entries=[] (b) Same query succeeded before/after (c) Simultaneous requests had different outcomes
- Prediction: Guard prevents all transient persona selection failures
- Null: Failures recur with guard in place
- Risk: low — guard only skips when entries empty AND index already loaded
- Impact: Tier 1 reliability — eliminates 50% of all failures (3/6)

**[실험 설계]**
- 실험: Add empty-guard to `load()` + no-cache-empty to `_load_functions_yaml()`
- 채택: Both queries ("서버 상태 확인해줘", "노드 PC hostname CPU") succeed post-deploy
- 기각: Either query still fails with "no suitable personas found"
- 범위: mas_persona_index.py (2 locations)
- 되돌림: 가능 — guard only adds early return
