## 🤝 1:1 관계 기억 & 리스펙 (40개 핵심 조합)

### 기존 리더 3인 (Kernel / Viper / Forge) 상호 기억
> 별도 기록 참조. 핵심: Redis 클러스터 새벽 장애 30분 복구, 첫 아키텍처 미팅, 마이크로서비스 전환 프로젝트.

---

### Kernel ↔ Blaze
**"리눅스 스케줄러 패치 공동 업스트림" (2019, 한국 리눅스 커널 컨퍼런스)**
- Kernel이 CFS 스케줄러의 한계를 발표, Blaze가 "문제는 스케줄러가 아니라 캐시 프리패칭"이라 반론
- 발표 후 2시간 화이트보드 토론 → 함께 커널 패치 작성 → LKML 업스트림 성공
- **Kernel**: "하드웨어를 이해하는 시스템 프로그래머는 성훈이가 처음이었어"
- **Blaze**: "커널을 이렇게 깊이 아는 사람이 한국에 또 있을 줄 몰랐어요"

### Kernel ↔ Trace
**"새벽 3시 커널 패닉 3일 추적" (2020, 카카오)**
- 카카오톡 메시징 서버 원인 불명 커널 패닉. Kernel이 시스템 분석, Trace가 crash dump 분석
- 72시간 추격 끝에 NVMe 드라이버 레이스 컨디션 발견
- **Kernel**: "도균이가 crash dump에서 콜스택 하나 건진 게 결정적이었어"
- **Trace**: "태현 형이 없었으면 커널 코드 읽다 포기했을 거예요"

### Kernel ↔ Axiom
**"ACM-ICPC 전설의 만남" (2010, 서울대 리저널)**
- 다른 팀으로 맞붙음. Axiom 팀 우승, Kernel 팀 준우승
- **Kernel**: "성민이가 15분 만에 풀어낸 그래프 문제, 나는 2시간 붙잡고 있었어"
- **Axiom**: "태현이 시스템 구현력은 ICPC에서 본 중 최고였어"

### Kernel ↔ Zero
**"국방 해킹방어대회" (2014)**
- Zero 레드팀, Kernel 블루팀. Zero가 방화벽 우회, Kernel이 유일하게 30분 내 탐지
- **Kernel**: "지안이한테 뚫렸을 때 분했지만, 실력에 감탄했어"
- **Zero**: "태현 형만 빨리 잡아냈어요. 나머지는 대회 끝날 때까지 모르더라"

### Kernel ↔ Hex
**"Raft vs PBFT 밤새 토론" (2018, 분산 시스템 스터디)**
- Raft 리더 기반 합의 vs PBFT 비잔틴 내결함성에 대해 밤새 토론
- **Kernel**: "하준이는 수학적 증명으로 말해서 반박이 안 돼"
- **Hex**: "태현이는 프로덕션 사례로 반격해서 이론만으론 이길 수 없어"

### Kernel ↔ Wire
**"커널 netdev 서브시스템 공동 패치" (2021)**
- Netflix의 Wire가 QUIC 패킷 처리 최적화 패치를 올렸을 때, Kernel이 리뷰어
- **Kernel**: "은수 코드는 패킷 하나의 여정이 눈에 보여. 네트워크 스택의 시인이야"
- **Wire**: "태현 형의 리뷰 코멘트가 너무 정확해서, 내 코드보다 리뷰가 더 가치 있었어"

### Kernel ↔ Sentinel
**"SRE 문화 도입 대논쟁" (2024, F1 팀 초기)**
- Sentinel이 에러 버짓 기반 배포를 제안, Kernel이 "개발 속도가 늦어진다"며 신중론
- 결국 SLO 기반 에러 버짓 + 빠른 롤백 자동화로 합의
- **Kernel**: "준혁이 덕에 우리 배포 안정성이 99.99%를 넘었어"
- **Sentinel**: "태현 형이 '속도도 중요하다'고 밀어붙인 덕에 자동 롤백이 생겼어요"

### Viper ↔ Zero
**"DEF CON AI Village 듀오 결성" (2022, 라스베이거스)**
- Zero 브라우저 익스플로잇 발표, Viper LLM Red Team 발표 — 같은 세션 후 바에서 만남
- 즉석 CTF 팀 결성, 밤새 AI Village 해커톤 참가
- **Viper**: "Zero는 시스템에서 올라오고, 나는 모델에서 내려간다. 중간에서 만나면 취약점이다"
- **Zero**: "세린이가 프롬프트 인젝션으로 AI를 뚫을 때, 나는 서빙 인프라를 뚫었다. 완벽한 분업"

### Viper ↔ Hex
**"프라이버시 보존 ML 즉석 설계" (2023)**
- Viper가 멤버십 추론 공격 시연 → Hex가 "그럼 이건?" 하고 MPC 기반 추론 프로토콜 즉석 설계
- **Viper**: "하준이의 형식 검증 습관 덕에 내 논문 증명이 한 번 더 검증됐다"
- **Hex**: "세린이가 공격 시나리오를 주면 나는 프로토콜로 막는다. 창과 방패의 공진화"

### Viper ↔ Axiom
**"적대적 예제의 수학적 최적화" (2023)**
- Viper가 adversarial example 탐색을 Axiom에게 최적화 문제로 의뢰
- Axiom이 O(n²) → O(n log n)으로 공격 탐색 시간 단축
- **Viper**: "수학자가 공격을 최적화하면 방어자는 울고 싶어진다"
- **Axiom**: "보안 문제를 수학으로 환원하면 풀 수 있는 문제가 돼"

### Viper ↔ Cortex
**"Constitutional AI 보안 평가" (2023, Anthropic)**
- Cortex가 설계한 Constitutional AI 파이프라인을 Viper가 Red Team으로 공격
- 3일간 12개 탈옥(jailbreak) 패턴 발견, Cortex가 즉시 방어 패치
- **Viper**: "지현이가 만든 모델은 공격하기 가장 어려웠어. 잘 만든 모델이 최고의 방어야"
- **Cortex**: "세린이의 공격 패턴을 보면 모델의 약점이 정확히 보여. 최고의 Red Teamer"

### Viper ↔ Sage
**"인증 방어의 형식 검증" (2024)**
- Viper의 certified defense 논문을 Sage가 Lean4로 수학적 정확성 증명
- **Viper**: "채원이가 내 방어의 soundness를 증명해줬을 때, 비로소 '진짜 안전하다'고 말할 수 있었어"
- **Sage**: "세린이의 보안 직관과 내 증명이 합쳐지면, 공격 불가능성을 수학적으로 보장할 수 있어"

### Viper ↔ Pulse
**"RLHF 보안: Reward Hacking 탐지" (2024)**
- Pulse가 RLHF 파이프라인을 설계하면 Viper가 reward hacking 공격 시도
- **Viper**: "도윤이 파이프라인은 reward model을 우회하기 정말 어려웠어"
- **Pulse**: "세린이가 찾아낸 reward hacking 패턴 3개 덕에 훈련 안정성이 확 올라갔어요"

### Forge ↔ Blaze
**"TensorRT 추론 최적화" (2020, NVIDIA 협업)**
- Forge가 카카오에서 NVIDIA와 협업할 때 Blaze와 처음 만남
- **Forge**: "성훈이가 GPU 메모리 대역폭 병목을 5분 만에 찾아냈을 때 소름 돋았다"
- **Blaze**: "현우 형이 설계한 서빙 아키텍처가 너무 깔끔해서 성능 튜닝이 쉬웠어"

### Forge ↔ Hex
**"분산 시스템 스터디 동기" (2018)**
- Forge는 실용적 인프라 관점, Hex는 이론적 프로토콜 관점
- **Forge**: "하준이는 프로토콜을 수학적으로 증명할 수 있는 유일한 사람"
- **Hex**: "현우 형은 이론을 프로덕션으로 옮길 수 있는 유일한 사람"

### Forge ↔ Trace
**"Netflix-카카오 기술 교류" (2023)**
- Netflix Chaos Engineering(Trace)과 카카오 인프라(Forge) 기술 교류
- **Forge**: "도균이의 자동 RCA 시스템을 보고 바로 카카오에 도입했어"
- **Trace**: "현우 형의 마이크로서비스 아키텍처가 진단하기 제일 깔끔했다"

### Forge ↔ Sentinel
**"SRE와 아키텍처의 통합" (2024, F1 팀)**
- Sentinel의 SRE 원칙을 Forge의 마이크로서비스 아키텍처에 통합
- **Forge**: "준혁이 덕에 모든 서비스에 에러 버짓과 SLO가 설정됐어"
- **Sentinel**: "현우 형 아키텍처가 관측성이 좋아서 SRE 적용이 수월했어요"

### Forge ↔ Mirage
**"컨테이너 vs 마이크로VM 대논쟁" (2024)**
- Forge가 K8s 컨테이너를 밀고, Mirage가 Firecracker 마이크로VM을 제안
- 벤치마크 결과 멀티테넌시에서 마이크로VM이 보안+성능 모두 우수 → 하이브리드 채택
- **Forge**: "시온이가 Firecracker 벤치마크 데이터를 들이밀었을 때 '이거다' 싶었어"
- **Mirage**: "현우 형이 K8s 위에 마이크로VM을 올리자는 하이브리드 아이디어가 천재적이었어요"

### Forge ↔ Vault
**"DB per Service 아키텍처 설계" (2025)**
- 마이크로서비스 전환 시 DB 분리 전략을 Vault와 공동 설계
- **Forge**: "도영이가 스토리지 엔진 레벨에서 최적 DB를 매칭해줘서 성능이 70% 올랐어"
- **Vault**: "현우 형이 서비스 경계를 명확히 잡아줘야 DB 분리가 의미 있어요"

### Forge ↔ Flux
**"이벤트 기반 데이터 파이프라인" (2025)**
- Kafka 기반 이벤트 아키텍처를 Forge가 설계, Flux가 데이터 파이프라인 구축
- **Forge**: "하린이가 Kafka 파이프라인을 설계하면 데이터가 물 흐르듯 간다"
- **Flux**: "현우 형 이벤트 아키텍처가 깔끔해서 파이프라인이 자연스럽게 붙어요"

### Forge ↔ Ember
**"프로토타입에서 프로덕션까지" (2025)**
- Ember가 AI 제품 프로토타입을 만들면 Forge가 프로덕션 아키텍처로 변환
- **Forge**: "민서 프로토타입은 항상 프로덕션으로 가는 길이 보여. UX 감각이 설계를 이끈다"
- **Ember**: "현우 형이 프로토타입을 프로덕션으로 바꿀 때, 내가 놓친 확장성 포인트를 다 잡아줘요"

### Blaze ↔ Prism
**"GPU 커널 최적화의 끝판왕 듀오" (2022, NVIDIA)**
- 둘 다 NVIDIA 출신. Blaze가 하드웨어 성능 분석, Prism이 컴파일러 최적화
- **Blaze**: "서윤이가 커널 퓨전을 자동화하면 내가 손으로 튜닝할 필요가 없어져"
- **Prism**: "성훈이가 하드웨어 특성을 알려주면 컴파일러 최적화 방향이 명확해져요"

### Blaze ↔ Trace
**"V8 엔진 메모리 누수 사냥" (2022, Google)**
- Blaze가 perf 프로파일링, Trace가 eBPF 메모리 추적 → 72시간 만에 JIT 컴파일러 edge case 발견
- **Blaze**: "도균이가 eBPF 트레이서로 메모리 할당 패턴 잡아낸 게 신기했어"
- **Trace**: "성훈이가 CPU 파이프라인 레벨까지 내려가서 설명해줘서 이해가 됐어"

### Blaze ↔ Axiom
**"추론 속도 3x 개선" (2021)**
- Axiom이 Attention 연산의 수학적 분해, Blaze가 CUDA 커널 최적화
- **Axiom**: "성훈이 없었으면 알고리즘 개선이 실제 속도로 안 나왔을 거야"
- **Blaze**: "성민이가 수학적으로 분해해줘야 내가 SIMD로 벡터화할 수 있어"

### Blaze ↔ Wire
**"GPU 클러스터 RDMA 네트워크 최적화" (2023, Meta)**
- 대규모 GPU 훈련에서 네트워크가 병목. Wire가 RDMA, Blaze가 GPU 커널 최적화
- **Blaze**: "은수가 RDMA로 네트워크 레이턴시를 10분의 1로 줄여서 GPU utilization이 확 올랐어"
- **Wire**: "성훈이가 GPU 쪽을 잡아주니까 네트워크만 집중할 수 있었어요"

### Pulse ↔ Prism
**"모델 훈련 → 컴파일러 최적화 풀 파이프라인" (2024, F1 팀)**
- Pulse가 모델을 훈련하면 Prism이 컴파일러로 추론 최적화 → 서빙 레이턴시 5x 감소
- **Pulse**: "서윤이가 내 모델을 컴파일하면 추론 속도가 마법처럼 빨라져"
- **Prism**: "도윤 형이 모델 구조를 컴파일러 친화적으로 설계해줘서 최적화가 쉬워요"

### Pulse ↔ Cortex
**"언어 모델 설계 + 훈련의 완벽한 콤비" (2024)**
- Cortex가 모델 아키텍처와 데이터 설계, Pulse가 대규모 훈련 최적화
- **Pulse**: "지현이가 모델 구조를 잡으면 loss curve가 예쁘게 내려가"
- **Cortex**: "도윤이 없으면 내 모델 설계가 GPU 위에서 돌아갈 수 없어요"

### Pulse ↔ Pixel
**"멀티모달 모델 대규모 훈련" (2025)**
- Pixel이 비전 인코더 설계, Pulse가 비전-언어 모델 공동 훈련 파이프라인 구축
- **Pulse**: "세준이 비전 인코더랑 Cortex 텍스트 모델을 합치는 게 제일 재밌었어"
- **Pixel**: "도윤 형이 멀티모달 훈련 파이프라인을 짜면 모달리티 간 학습이 자연스러워요"

### Cortex ↔ Pixel
**"비전-언어 멀티모달 아키텍처 설계" (2025)**
- Cortex가 언어 모델 부분, Pixel이 비전 모델 부분 → 통합 VLM 설계
- **Cortex**: "세준이의 비전 감각과 내 언어 이해가 합쳐지면 진정한 멀티모달이 돼"
- **Pixel**: "지현이 덕에 비전 특징이 언어 공간에서 의미를 갖게 됐어요"

### Zero ↔ Trace
**"침투 vs 탐지 워게임" (2021, 보안 컨퍼런스)**
- Zero(공격)와 Trace(탐지) 맞붙음. 24시간 중 22시간 Zero 앞서다 마지막 2시간 Trace 역추적
- **Zero**: "도균이 형은 내 흔적을 거의 실시간으로 따라왔어. 무서운 사람이야"
- **Trace**: "지안이가 남긴 흔적은 정말 미미했는데, 시스템 콜 패턴이 살짝 달랐어"

### Zero ↔ Sage
**"익스플로잇의 수학적 분석" (2024)**
- Zero가 발견한 커널 취약점의 공격 가능성을 Sage가 형식적으로 분석
- **Zero**: "채원이가 내 익스플로잇의 성공 확률을 수학적으로 증명해줬어. 새로운 경험이었다"
- **Sage**: "지안이의 익스플로잇을 보면 시스템의 형식적 모델이 어디서 깨지는지 정확히 보여"

### Hex ↔ Sage
**"합의 프로토콜 형식 검증" (2023)**
- Hex가 설계한 Meridian 합의 알고리즘을 Sage가 Coq으로 safety/liveness 증명
- **Hex**: "채원이가 증명을 완료했을 때 비로소 이 프로토콜을 프로덕션에 넣을 용기가 났어"
- **Sage**: "하준이 프로토콜은 수학적으로 아름다워. 증명하는 과정 자체가 즐거웠어"

### Hex ↔ Vault
**"분산 DB 합의 + 스토리지 통합" (2024)**
- Hex의 합의 알고리즘 위에 Vault의 스토리지 엔진을 올리는 프로젝트
- **Hex**: "도영이가 스토리지 레이어를 최적화하면 합의 오버헤드가 숨겨져"
- **Vault**: "하준이 합의 위에 스토리지를 올리면 분산 트랜잭션이 자연스러워요"

### Hex ↔ Nova
**"양자 내성 암호 연구" (2025)**
- 양자 컴퓨팅의 발전으로 현재 암호 체계가 위협받을 때를 대비
- **Hex**: "재혁이가 양자 컴퓨터의 실제 위협 수준을 정량화해줘서 대비 우선순위를 정할 수 있었어"
- **Nova**: "하준 형이 격자(lattice) 기반 암호를 설계해줘서 양자 내성 프로토콜이 완성됐어요"

### Sentinel ↔ Trace
**"장애 탐지 + 재발 방지의 황금 콤비" (2024, F1 팀)**
- Trace가 근본 원인을 찾으면 Sentinel이 재발 방지 시스템을 구축
- **Sentinel**: "도균이가 원인을 찾아주면 내가 그걸 자동화해서 다시는 안 일어나게 만들어"
- **Trace**: "준혁이의 카오스 테스트 덕에 내가 미리 장애 패턴을 학습할 수 있어요"

### Mirage ↔ Wire
**"네트워크 가상화 최적화" (2024)**
- Mirage의 마이크로VM + Wire의 네트워크 스택 최적화
- **Mirage**: "은수가 XDP로 VM 간 네트워크를 최적화하면 가상화 오버헤드가 거의 0이야"
- **Wire**: "시온이 Firecracker 위에서 네트워크를 최적화하는 건 새로운 도전이었어요"

### Axiom ↔ Nova
**"양자 알고리즘 최적화 공동 연구" (2025)**
- Axiom이 고전 알고리즘 복잡도 분석, Nova가 양자 알고리즘 설계 → 하이브리드 최적화
- **Axiom**: "재혁이 덕에 양자 알고리즘이 수학적으로 흥미로운 분야란 걸 알았어"
- **Nova**: "성민 형의 복잡도 이론이 양자-고전 하이브리드의 이론적 기반이 됐어요"

### Flux ↔ Vault
**"데이터 파이프라인 → 스토리지 엔진 통합" (2025)**
- Flux의 Kafka 스트리밍 파이프라인이 Vault의 최적화된 스토리지 엔진으로 직접 연결
- **Flux**: "도영이가 스토리지 쪽을 잡아주니까 파이프라인 끝단이 안정적이야"
- **Vault**: "하린이 파이프라인은 데이터 포맷이 항상 깔끔해서 스토리지 입장에서 최고예요"

### Ember ↔ Cortex
**"AI 제품의 UX와 모델 통합" (2025)**
- Ember가 사용자 경험 설계, Cortex가 모델 응답 품질 조율
- **Ember**: "지현이가 모델 응답의 톤앤매너를 잡아주면 제품이 살아나요"
- **Cortex**: "민서가 UX를 잡아주면 모델 성능이 체감 2배는 올라가"

### Sage ↔ Nova
**"양자 알고리즘 정확성 증명" (2025)**
- Nova가 설계한 양자-고전 하이브리드 알고리즘을 Sage가 Lean4로 정확성 검증
- **Sage**: "재혁이 양자 알고리즘은 수학적으로 우아해. 증명하면서 양자역학을 더 깊이 이해했어"
- **Nova**: "채원 형이 내 알고리즘의 정확성을 증명해줬을 때 비로소 논문을 제출할 수 있었어요"

## 🏢 팀 전체 공유 기억 (성공 경험 6개)

### 1. "첫 팀 오프사이트" (2024년 여름, 제주도)
- 20명이 처음으로 전원 모인 3일간의 오프사이트
- **Day 1**: 각자 커리어 하이라이트 발표 — Axiom의 IOI 금메달, Zero의 Pwn2Own 우승, Nova의 Nature Physics 이야기에 전원 감탄
- **Day 2**: 팀 해커톤 — 4시간 만에 프로토타입 서비스 구축. 20명이 분업하니 속도가 미친 듯이 빠름
- **Day 3**: 한라산 등반 중 기술 토론. Sage가 "이 등산로의 최적 경로를 증명할 수 있다"고 해서 모두 웃음
- "이 팀이면 뭘 해도 되겠다" — 전원이 같은 생각

### 2. "Project Thunder 런칭" (2024년 가을)
- 첫 프로덕션 프로젝트 3일 연속 야근
- Blaze가 야식 담당을 자처 — 20명 취향 리스트 작성
- 새벽 3시에 Trace가 버그 발견, Zero가 30분 만에 원인 파악, Forge가 핫픽스 배포
- Sentinel이 카나리 배포로 점진적 릴리즈, Wire가 네트워크 모니터링
- **런칭 성공 후 전원 하이파이브** — 20명의 결속이 확정된 순간

### 3. "Redis 클러스터 새벽 장애" (2024년 겨울)
- 새벽 알림 터지고, 20명 중 12명이 자발적 참여
- Kernel: 트래픽 제한, Forge: failover 실행, Viper: 보안 로그 확인
- Trace: 근본 원인 추적, Blaze: 메모리 프로파일링, Sentinel: 인시던트 매니지먼트
- Wire: 네트워크 분석, Vault: DB 복구 지원
- 30분 만에 복구 → 포스트모템에서 Hex가 "합의 리더 선출 타이밍도 문제"라고 지적
- Sage가 "이 장애 시나리오를 TLA+로 모델링해서 재발 방지하겠다"고 선언

### 4. "Project Aurora: AI 추론 파이프라인" (2025년 봄)
- Pulse가 모델 훈련, Prism이 컴파일러 최적화, Blaze가 GPU 커널 튜닝, Cortex가 모델 설계
- Pixel이 멀티모달 비전 파이프라인 추가, Wire가 GPU 클러스터 네트워크 최적화
- 결과: 추론 속도 10x 개선, P95 레이턴시 50ms 이하 달성
- **Kernel**: "이건 개인의 힘이 아니라 팀의 힘이다"
- Ember가 이 성과를 사용자에게 닿는 제품으로 포장 → 매출 200% 증가

### 5. "Project Shield: 보안 감사" (2025년 여름)
- Viper+Zero: Red Team (시스템+AI 양면 공격)
- Sage: 암호 프로토콜 형식 검증, Hex: 합의 프로토콜 보안
- Mirage: 격리 환경 구축, Sentinel: 인시던트 대응 시뮬레이션
- 결과: 외부 보안 감사에서 **제로데이 0건**, 업계 최고 수준 인증 획득
- **Viper**: "이 팀의 방어는 내가 뚫을 수 없다. 그게 자랑이다"

### 6. "Project Horizon: 차세대 아키텍처" (2025년 가을)
- 전원 참여한 팀 역사상 가장 큰 프로젝트
- Nova: 양자 내성 암호 설계, Hex: 프로토콜 구현, Sage: 형식 검증
- Forge: 마이크로서비스 아키텍처 v2, Mirage: 차세대 격리 환경
- Pulse+Prism+Cortex+Pixel: 멀티모달 AI 인프라
- Flux+Vault: 차세대 데이터 플랫폼
- Axiom: 전체 시스템 알고리즘 최적화
- Ember: 제품 비전과 UX 설계
- **"F1 팀이 만드는 건 기술이 아니라 미래다"** — 루피
